# Data Engineering: ETL Pipeline & DWH Architecture (US Flights Analysis)

Данный проект представляет собой техническую реализацию хранилища данных (DWH) для аналитической обработки информации об авиаперевозках США. Архитектура ориентирована на обеспечение качества, консистентности данных и автоматизацию жизненного цикла.

### Архитектурные слои (ELT/ETL)
Проектирование хранилища выполнено по многослойной структуре, обеспечивающей масштабируемость и дедупликацию:

* **ODS (Operational Data Store):** Слой загрузки «сырых» данных из S3-хранилища.
* **STG (Staging):** Первичная предобработка данных, приведение форматов, перевод времени в UTC.
* **DDS (Data Delivery Store):** Слой обогащения и дедупликации с использованием стратегии `on conflict` для обеспечения целостности данных.
* **DM (Data Mart):** Финальные витрины данных, оптимизированные для аналитических запросов.

### Оркестрация и стек
* **Оркестрация:** Полная автоматизация процессов реализована через Apache Airflow (система DAG-ов с мониторингом отказов и триггерами).
* **Стек:** PostgreSQL (основное хранилище), ClickHouse (аналитическая СУБД для BI), Apache Airflow, S3.
* **Интеграция:** Реализована интеграция PostgreSQL и ClickHouse, что обеспечивает работу с данными в реальном времени без избыточного дублирования.

### Основные технические достижения
* **Масштабируемость:** Система построена на инкрементальной загрузке, что позволяет обрабатывать новые данные без полной пересборки хранилища.
* **Аналитическая готовность:** Созданы универсальные витрины для анализа задержек, отмен и успешных рейсов, позволяющие строить любые срезы в Yandex DataLens.
* **Надежность:** Реализована защита от дублей и логика обработки ошибок на уровне Airflow-пайплайнов.
